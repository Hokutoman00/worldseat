proof layer
Consistency Witness — Jepsen-lite linearizability check
Reads the committed per-seat history — DSQL op_log and the append-only DynamoDB sales_naive ledger — and computes doubleSells = Σ max(0, confirmed−1). Verdict is calculated from real logs, never hard-coded. Honesty guard: oversold is counted from the ledger (ground truth), not from what the app hoped happened.
Stack: Next.js 14.2 / React 18 on Vercel · one AWS database per mode (Aurora DSQL or DynamoDB Global Tables) · pg + @aws-sdk/dsql-signer + @aws-sdk/lib-dynamodb.
Live-proven: DSQL 72 concurrent buyers → oversold = 0 · naive 72 concurrent buyers → oversold = 29 (4 seats double-sold) — identical application code, identical concurrency, different database.
